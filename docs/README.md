# Ephemeral Board

Miro / FigJam ライクなリアルタイム共同付箋ボードアプリケーション。
ローカルネットワーク上で動作し、HTTP トンネルを通じて社内メンバーと共同ワークできる。

Markdown でボード状態を保存・復元できるため、セッションをまたいだ持ち越しにも対応。

## クイックスタート

### 前提条件

- Node.js v20 以上

### インストールと起動

```bash
cd ephemeral_board
pnpm install
pnpm dev
```

ブラウザで http://localhost:3000 を開く。

### 社内メンバーと共有する（pnpm share）

```bash
pnpm share              # 既定で cloudflared の接続診断ログを stderr に逐次出力
pnpm share --qr         # トンネル URL の QR コードもターミナルに描画
pnpm share --quiet      # 診断ログを抑止して URL バナーのみ表示
```

これだけで以下が一気に立ち上がる:

1. クライアントを本番ビルド（`pnpm build` 相当）
2. サーバーを起動
3. Cloudflare Tunnel を起動（初回のみ `cloudflared` バイナリを自動ダウンロード）
4. `Local` URL と `Tunnel` URL（`*.trycloudflare.com`）をターミナルに表示
5. `--qr` (または `-q`) を付けた場合のみ、トンネル URL の QR コードを描画（スマホで即読み取り可）

起動後、ターミナル上で以下のキーが効く:

| キー | 動作 |
|------|------|
| `b` | 既定ブラウザでトンネル URL を開く |
| `c` | トンネル URL をクリップボードへコピー（Linux は `xclip` / `wl-copy` が必要） |
| `q` / `Ctrl+C` | サーバーとトンネルを順にクリーン停止 |

#### コマンドラインオプション

| フラグ | 動作 |
|--------|------|
| `--qr` / `-q` | トンネル URL の QR コードをターミナルに描画する |
| `--quiet` / `-s` | cloudflared 診断ログを抑止し、URL バナーだけ表示する |

トンネル URL が出ない・接続が切れる等のトラブル時は、stderr に出る cloudflared 診断ログ（既定 ON）が手がかりになる。診断ログのフォーマット・自前 cloudflared での起動・本番風起動・Named Tunnel への移行などは [DEVELOPMENT.md](https://github.com/Trippy3/ephemeral_board/blob/main/docs/DEVELOPMENT.md) を参照。

## Claude Code スキルでボードを自動生成する

[Claude Code](https://claude.com/claude-code) を使っている場合、議事録・ブレストメモ・要件リスト・記事要約などの雑然とした情報を、付箋・色分け・フレーム・矢印で整理されたボードとして **自動生成** できる。

リポジトリの `.claude/skills/create-sticky-board/` に `create-sticky-board` スキルを同梱しており、Claude Code がプロジェクト直下で起動した際に自動で読み込まれる。

### 使い方

Claude Code のセッション内で `/create-sticky-board` をスラッシュコマンドとして呼び出し、ボード化したい情報を渡す:

```text
/create-sticky-board 以下の議事録をボードにまとめてください。
- ターゲットは社内エンジニア
- ペインポイント: ドキュメントが散らばっている / 検索しても古い情報が出る
- アイデア: Slack 常駐の Q&A bot / ドキュメント鮮度のスコア化
- アクション: PoC を 2 週間 / 利用者ヒアリング 5 名
```

スキルは次を自動で行う:

1. 入力テキストの構造を読み取り、カテゴリ分けを決定（前提 / 課題 / アイデア / アクションなど）
2. レイアウト方針を選択（カテゴリ別グリッド / フローチャート / マインドマップ / マトリクス / タイムライン）
3. 凡例フレームと色分けを設計（10 色のうち 3〜5 色をカテゴリへ割当）
4. 付箋・フレーム・矢印を含む 1 つの `.md` ファイルを生成

生成された `.md` を **Import MD** から読み込めば、整理済みのボードがそのまま復元される。

### 向いているユースケース

- 議事録・ブレストメモを付箋として並べたい
- 論点・課題・アイデアを色とフレームで分類したい
- 因果関係・依存関係・フローを矢印でつなげて見える化したい
- Web 記事や長文を要約してマインドマップ化したい

逆に、既にエクスポート済みの `.md` を編集したい場合や、単にテキスト要約だけが欲しい場合は通常の編集で十分。

### 出力フォーマットとカスタマイズ

スキルが生成する `.md` は本リポジトリの Markdown インポート形式（YAML フェンス）に準拠している。
スキル本体・フォーマット仕様・サンプル出力は `.claude/skills/create-sticky-board/` 配下を参照:

- `SKILL.md` — レイアウト規則・色の使い分け・ID 発行ルールなどの設計方針
- `references/example_output.md` — 完全な出力サンプル
- `references/format_spec.md` — フィールド・バリデーション仕様

Markdown インポート形式そのものの詳細は [DEVELOPMENT.md — Markdown 出力フォーマット仕様](https://github.com/Trippy3/ephemeral_board/blob/main/docs/DEVELOPMENT.md#markdown-出力フォーマット仕様) を参照。

## 操作方法

URL を開くと名前入力ダイアログが表示される。名前を入力して「Join」をクリック（Enter キーでも可）すると、ボードに参加して共同作業が始まる。

URL パスがそのままボード ID になる。異なる URL にアクセスすれば独立したボードが自動作成される。

| URL | ボード ID |
|-----|----------|
| `http://localhost:3000/` | `default` |
| `http://localhost:3000/sprint-retro` | `sprint-retro` |
| `http://localhost:3000/brainstorm-2026` | `brainstorm-2026` |

### 付箋

#### 作成・編集

| 操作 | 方法 |
|------|------|
| 付箋を作成 | ボードの空白部分をダブルクリック |
| テキスト編集 | 付箋内のテキスト部分をクリックして直接入力 |
| 太字 | 編集中に Ctrl+B (Mac は ⌘+B) で選択範囲をトグル |
| 文字揃え | 付箋にホバー → ⯇ / ≡ / ⯈ ボタン |
| フォントサイズ | 付箋にホバー → A− / A+ ボタン (12 / 14 / 18 / 24 px) |
| 移動 | 付箋のヘッダーまたはテキスト以外の余白をドラッグ |
| リサイズ | 付箋の右下角をドラッグ |
| 色を変更 | 付箋にホバー → 🎨 ボタンをクリック → 色を選択 |
| 削除 | 付箋にホバー → ✕ ボタン、または選択中に Delete キー |
| 削除取り消し | Ctrl+Z (自分が削除した最大 20 件) |
| コピー / 貼付 | 選択して Ctrl+C → カーソル位置で Ctrl+V |

#### 選択

| 操作 | 方法 |
|------|------|
| 単一選択 | 付箋をクリック |
| 追加選択 | Shift + 付箋クリック |
| 矩形選択 | 空白を左ドラッグ |
| 矩形で追加選択 | Shift + 空白を左ドラッグ |
| 選択解除 | Esc / 空白を単純クリック |
| 一括移動 | 選択中の任意の付箋をドラッグ |
| 一括削除 | Delete キー |

#### 矢印で結ぶ

専用モードに切り替える必要はない。

1. 始点にしたい付箋に**ホバー** → 上下左右の四辺中央に**青い●（edge anchor）**が表示される
2. 任意の ● から**ドラッグ**を開始 → **全ての付箋の●が表示**される（ドロップ先候補）
3. 終点にしたい付箋の**●**にドロップ（緑色にハイライト）→ そのアンカー同士を結ぶ矢印が描画される
4. 線は両端のアンカー（辺中央）に**固定**される。付箋を動かしてもそのアンカーに追従する  
   ※ 付箋本体（●以外）にドロップした場合は、ドロップ位置に最も近い辺が自動で選ばれる

既存のコネクタをクリックするとミニメニューが開き、形状（直線 `━` / カクカク `⌐` / 弧 `⌒`）と線端（矢印 `→` / 線 `—`）を独立に切り替え、`✕` で削除できる。

どこにもドロップしない、または同じ付箋の上でドロップするとキャンセルされる。接続元/接続先の付箋を削除すると、対応するコネクタは自動的に消える。

#### フレームで囲む

1. ツールバーの ⬜ ボタンをクリック (または `F` キー) → フレーム描画モード
2. ボードの空白を左ドラッグして矩形を描画
3. 描画後、自動でモード解除
4. タイトル編集 / ホバー時の ✕ で削除 / 右下角でリサイズ / 本体ドラッグで移動

#### 作成時の色を選ぶ

ツールバー左側のカラーパレットから色を選んでおくと、次に作成する付箋がその色になる。10 色から選択可能 (Yellow / Red / Green / Blue / Purple / Orange / Pink / White / Gray / Black)。白付箋は枠線付きで背景に溶け込まず、黒付箋は文字色が自動で白に切り替わる。

### ボード全体

#### パン・ズーム

| 操作 | 方法 |
|------|------|
| パン（基本） | **右クリック+ドラッグ** または **中クリック+ドラッグ** |
| パン（一時切替） | **Space を押しながら左ドラッグ** |
| ズームイン/アウト | マウスホイール、またはツールバーの +/- ボタン |
| ズームリセット | ツールバーの `100%` ボタン |
| モード解除 / 選択解除 | Esc |

ズームはマウスカーソルの位置を中心に拡大/縮小される（0.2x〜3.0x）。ボード内では右クリックメニューは抑制されている（パン操作のため）。

#### 他のユーザーの存在確認

- ツールバー右側にユーザーの頭文字アイコンが表示される
- ボード上に他のユーザーのカーソル（色付き矢印 + 名前ラベル）がリアルタイムで表示される

#### モバイル端末（タッチ）での操作

PC のショートカットや右クリックは使えないため、ジェスチャベースに置き換わっている。

| 操作 | タッチ |
|------|--------|
| パン | **1 本指でドラッグ**（空白部分） |
| ズーム | **2 本指でピンチ** + ピンチ中の 2 本指パン |
| 付箋を作成 | 空白を**長押し** (約 0.5 秒)、または**ダブルタップ** |
| 付箋を選択 | 付箋をタップ |
| 選択解除 | 空白を 1 回タップ |
| 付箋を移動 | 付箋を 1 本指でドラッグ |
| アクションボタン表示 | 付箋を選択中（`B` / 整列 / フォントサイズ / 🎨 / ✕ などが見える） |
| 付箋テキスト編集 | 付箋本文をタップ → ソフトキーボードで入力 |
| コネクタ作成 | 付箋を選択して edge anchor (●) を表示 → ● から別の付箋へドラッグ |
| 付箋リサイズ | 選択中の右下角ハンドルをドラッグ |
| フレーム描画 | ⬜ ボタン → 空白を 1 本指でドラッグ |

タッチ端末ではタッチターゲットが自動的に拡大される（edge anchor / アクションボタン / リサイズハンドル等）。

**制限事項**: 矩形選択は 1 本指ドラッグがパンに割り当てられているため非対応（複数選択は付箋を 1 つずつタップ）。Undo (Ctrl+Z) / コピペ / 太字 (Ctrl+B) などのキーボードショートカットはソフトキーボードでも発火しないため PC 専用。付箋削除は ✕ ボタン経由になる。

## Markdown のエクスポート / インポート

ツールバーの **Export MD** でボードの全状態を Markdown ファイルとしてダウンロードできる。
**Import MD** で同形式のファイルを読み込み、ボードを完全に復元できる。

出力ファイルには YAML front-matter・再インポート用のデータブロックが含まれる。
詳細なフォーマット仕様は [DEVELOPMENT.md — Markdown 出力フォーマット仕様](https://github.com/Trippy3/ephemeral_board/blob/main/docs/DEVELOPMENT.md#markdown-出力フォーマット仕様) を参照。

### インポート時の挙動

1. **Import MD** をクリック → ファイルを選択
2. 確認ダイアログに付箋・コネクタ・フレームの件数が表示される
3. 「置き換える」を押すと現在のボードが**完全に置き換えられ**、参加中の全クライアントに即座に反映される
4. 不正な YAML や未知のスキーマは事前にバリデーションで弾かれる（エラー時はダイアログ前にアラート表示）

事前にエクスポートしておくと安全（インポートは置換のみで取り消し不可。詳細は末尾の「注意点」）。

### 制限と安全策

- ファイルサイズ上限: 1 MB
- 要素数上限: 1000（付箋 + コネクタ + フレームの合計）
- 不存在の付箋を参照するコネクタは自動的にドロップ
- 付箋テキストはクライアント (DOMPurify) とサーバー両方でサニタイズ。許可タグは `<b> <strong> <br> <div> <span> <p>`、許可属性は制限付き `style` のみ（`text-align` / `font-size`）

## 注意点

- **永続化なし**: サーバー再起動で全データが消える（仕様。MD エクスポート/インポートで対応）
- **自動クリーンアップ**: 最終操作から 24 時間経過したボードは自動削除される
- **認証なし**: ワークショップ・レトロ・ブレストといった一時的な共同作業を想定したアプリケーションで、機密情報を扱うことは想定していない。URL を知っていれば誰でもアクセスできる。情報を堅牢に扱いたい場合は、Named Tunnel + Cloudflare Access など別アプローチを各自で構築してください
- **テキスト競合**: 同一付箋の同時編集は LWW（後から書いた方が勝つ）
- **Undo のスコープ**: 自分の削除のみ・最大 20 件
- **インポートは置換のみ**: 既存ボードへのマージは非対応
- **最大規模**: DOM ベースのため、1 ボード数百枚程度が実用上限
